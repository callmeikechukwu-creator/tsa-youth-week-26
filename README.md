# TSA Youth Week 2026

**The Salvation Army IBCC Youth Fellowship — Youth Retreat 2026 Showcase**

A Node.js + Express + EJS web app with an admin dashboard, event registration, contact form, and Google Analytics integration.

---

## Tech Stack

| Layer | Technology |
|-------|-----------|
| Runtime | Node.js |
| Framework | Express 4 |
| Templating | EJS |
| Sessions | express-session + session-file-store |
| Analytics | Google Analytics 4 Data API |
| Hosting | Render |

---

## Local Development

### 1. Clone & install

```bash
git clone https://github.com/<your-username>/tsa-youth-week-26.git
cd tsa-youth-week-26
npm install
```

### 2. Configure environment

```bash
cp .env.example .env
```

Edit `.env` and fill in:

| Variable | Description |
|----------|-------------|
| `SESSION_SECRET` | Random 64-char string — run `node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"` |
| `ADMIN_EMAIL` | Admin login email |
| `ADMIN_PASSWORD` | Admin login password |
| `GA_TRACKING_ID` | GA4 Measurement ID (optional) |
| `GA_PROPERTY_ID` | GA4 Property ID (optional) |
| `GA_SERVICE_ACCOUNT_JSON` | Service account JSON string (optional) |

### 3. Run

```bash
npm run dev    # dev mode with hot reload
npm start      # production mode (also runs build first)
```

App runs at **http://localhost:3000**  
Admin panel at **http://localhost:3000/admin**

## Deploying to GitHub

```bash
# Inside the project folder
git init
git add .
git commit -m "Initial commit"

# Create repo on github.com first, then:
git remote add origin https://github.com/<your-username>/tsa-youth-week-26.git
git branch -M main
git push -u origin main
```

> **Important:** `.env`, `data/`, `_dev/`, and `service-account.json` are in `.gitignore` — they will NOT be pushed. Never commit secrets.

## Deploying to Render

### Step 1 — Connect repo

1. Go to [render.com](https://render.com) → **New → Web Service**
2. Connect your GitHub account and select the `tsa-youth-week-26` repo
3. Render will auto-detect `render.yaml` and pre-fill settings

### Step 2 — Set environment variables

In the Render dashboard → **Environment**, add these manually (marked `sync: false` in render.yaml):

| Variable | Value |
|----------|-------|
| `ADMIN_EMAIL` | your admin email |
| `ADMIN_PASSWORD` | a strong password |
| `GA_TRACKING_ID` | your GA4 ID (optional) |
| `GA_PROPERTY_ID` | your GA4 property ID (optional) |
| `GA_SERVICE_ACCOUNT_JSON` | paste full service account JSON as one line (optional) |

`SESSION_SECRET` is auto-generated by Render — no action needed.

### Step 3 — Deploy

Click **Create Web Service**. Render will:
1. Run `npm install`
2. Run `npm start` → which triggers `prestart` → `build:css` + `build:js`
3. Start the server on port 10000

### ⚠️ Free Plan — Data Persistence Warning

The free Render plan uses an **ephemeral filesystem**. This means:

- Registrations, contacts, and sessions stored in `data/` **will be wiped on every deploy or restart**
- Admin accounts are re-seeded from env vars on each start (safe)

**Fix:** Upgrade to the Render **Starter plan** ($7/mo) and uncomment the `disk:` block in `render.yaml`. This mounts a persistent disk at `data/` so your JSON data survives restarts.
